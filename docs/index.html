WebLogic Image Tool

Oracleは、WebLogic Serverを使用して重要なワークロードを実行したり、それらのワークロードをクラウドに移行したり、アプリケーション・デプロイメントのライフサイクルを簡略化および高速化する方法を見つけています。 DockerやKubernetesなどの業界標準を採用することで、WebLogicはクラウド・インフラストラクチャで中立に実行されるようになりました。 WebLogic Serverのコンテナ・イメージの作成を簡略化および自動化するために、このオープン・ソースのWebLogic Image Tool (WIT)を提供しています。

WITでは、JDKおよびWebLogic Serverのインストールを使用して新しいLinuxベースのイメージを作成し、オプションで、アプリケーションを使用してWebLogicドメインを構成するか、WebLogic Serverパッチを適用するか、既存のイメージを更新します。

イメージ・ツールを使用して、次のことができます:

  • コンテナ・イメージを作成し、リクエストされたJavaおよびWebLogicソフトウェアをインストールします。 また、イメージ内にWebLogicドメインを同時に作成できます。
  • 既存のイメージから既存のWebLogicドメインを使用して、新しいコンテナ・イメージを作成します。 Dockerイメージは、JDKおよびOracleミドルウェアのインストールで既存のイメージから起動することも、ドメインの移動の一部としてJDKおよびOracle Homeをインストールすることもできます。
  • 既存のイメージにWebLogicパッチを適用して、新しいコンテナ・イメージを作成します。 また、以前存在していなかった場合は、WebLogicドメインの作成、既存のドメインの更新またはアプリケーションのデプロイを実行できます。

詳細については、「イメージ・ツール」を参照してください。

サポートされているFusion Middleware (FMW)製品

WebLogic Image Toolを使用して、次のFMW製品イメージを作成、更新およびリベースします:

  • Oracle WebLogic Server (WLS)
  • Fusion Middleware Infrastructure (FMW)
  • Oracle BPM Suite for business process (SOA)
  • Oracle Service Bus (OSB)
  • Oracle Data Integrator (ODI)
  • Oracle Access Management (OAM)
  • Oracle Identity Governance (OIG)
  • Oracle Unified Directory (OUD)
  • Universal Content Management (UCM)
  • Oracle Managed File Transfer (MFT)
  • Oracle Internet Directory (OID)
  • Oracle WebCenter Sites (WCS)
  • Oracle Identity Manager (IDM)
  • Oracle WebCenter Content (WCC)
  • Oracle WebCenter Portal (WCP)
