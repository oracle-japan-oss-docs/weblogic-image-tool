WITへの貢献

Oracleは、このリポジトリへの貢献を誰からでも歓迎します。

プル・リクエストを送信してバグを修正するか、既存の機能を拡張する場合は、まず問題を開き、プル・リクエストを送信するときにその問題にリンクしてください。

可能な提出案について質問がある場合は、自由に問題を報告してください。

WebLogic Image Toolリポジトリへの貢献

プル・リクエストは、The Oracle Contributor Agreement (OCA)で実行できます。

プル・リクエストを受け入れるには、コミット・メッセージの最下部には、OCA署名リストに表示される名前と電子メール・アドレスを使用して次の行が必要です。

Signed-off-by: Your Name <you@example.org>

これは、次のようにコミットすることでプル・リクエストに自動的に追加できます:

$  git commit --signoff

OCAに署名したことを確認できるコミット担当者からのプル・リクエストのみを受け入れることができます。

プル・リクエスト・プロセス

  1. このリポジトリをフォークします。

  2. 変更を実装するためのブランチをフォークに作成します。 1234-fixなど、ブランチ名の一部として問題番号を使用することをお薦めします。

  3. 修正に必要な変更でドキュメントが更新されていることを確認します。

  4. ベース・イメージが変更された場合は、すべてのサンプルが更新されていることを確認します。

  5. ブランチの変更を1つのコミットに破棄します。

  6. プル・リクエストを送信します。

    a. プル・リクエストは空白のままにしないでください。

    b. 変更の目的を正確に説明し、変更のバリデート方法に関する簡単なステップを提供します。

    c. 作成した問題も必ず参照してください。

  7. プル・リクエストをマージする前にレビュー用に2-3人に割り当てます。
